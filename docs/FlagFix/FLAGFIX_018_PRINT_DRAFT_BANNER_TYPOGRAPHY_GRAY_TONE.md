# FlagFix_018 — Print Draft Banner Typography and Gray-Tone Design

**Status:** Open  
**Priority:** Low / Design polish  
**Scope:** Print/PDF review mode only  
**Area:** AXIS-NIDDHI static site print CSS  
**Related:** `#FlagFix_016`, `#FlagFix_017`, print review banner, review PDF traceability

---

## 1. Summary

The current `DRAFT / RASCUNHO` print review banner is functionally correct and visually elegant, but after printing dozens of review pages it became clear that the design can be refined.

The requested improvement is to make the banner feel more like an archival/typewritten review stamp:

- use a font that resembles a typewriter or technical machine print;
- change the banner border and text from dark/black styling to gray tones;
- use a darker gray for the border;
- use a slightly lighter gray for the text;
- preserve the sober, non-decorative, archival character of the current design.

This is a design polish issue, not a content or canonical issue.

---

## 2. Current Behavior

The print review banner appears only in print/PDF mode and provides traceability metadata such as:

- `DRAFT / RASCUNHO`
- review-copy notice;
- source reference;
- AXIS-NIDDHI page URL;
- canonical ID / PD#PN;
- explanatory archival note;
- color legend for glossary/audio term markings.

The current banner is useful and should be preserved conceptually.

---

## 3. Problem

The banner is now printed across many pages for human review. In repeated physical usage, the current visual treatment can feel slightly too strong or too “web/UI-like.”

For printed review packets, the banner should look more like an archival review seal or machine-generated annotation.

The desired effect:

> “This is clearly a review artifact generated by the preservation pipeline, not part of the original doctrinal text.”

---

## 4. Desired Design Direction

### Typography

Prefer a typewriter/monospace stack that is stable across operating systems and browsers:

```css
font-family:
  "Courier Prime",
  "Courier New",
  "Nimbus Mono PS",
  "Liberation Mono",
  monospace;
```

Notes:

- `Courier Prime` would be ideal if bundled later, but should not be required now.
- `Courier New` / `Liberation Mono` are safe defaults.
- Do not introduce remote font dependencies for print-critical output.
- Do not rely on Google Fonts or external CDN fonts.

### Color

Suggested gray-tone palette:

```css
border-color: #777;
color: #555;
```

Possible stronger archival alternative:

```css
border-color: #666;
color: #4f4f4f;
```

Possible softer alternative:

```css
border-color: #888;
color: #666;
```

Recommendation for first implementation:

```css
border: 1pt solid #777;
color: #555;
```

This keeps the banner legible while reducing visual dominance.

---

## 5. Suggested Surgical CSS Patch

Target files:

```text
pipeline/13-ssg/static/css/style.css
pipeline/13-static-site/css/style.css
```

Locate the print-only `.print-review-banner` rule and update only the typography/color values.

Conceptual patch:

```css
@media print {
  .print-review-banner {
    border: 1pt solid #777;
    font-family: "Courier Prime", "Courier New", "Nimbus Mono PS", "Liberation Mono", monospace;
    color: #555;
  }
}
```

If the current rule already contains padding, margin, overflow wrapping, box sizing, or max-width, preserve those properties.

Do not change the banner text, placement, URL logic, canonical ID logic, or color legend behavior in this FlagFix unless explicitly required by a later review.

---

## 6. Acceptance Criteria

A future patch for this FlagFix is acceptable if:

- screen view remains unchanged;
- banner remains hidden on screen;
- banner appears only in print/PDF mode;
- banner remains readable on A4 print;
- banner uses gray tones rather than black/dark UI styling;
- border is darker than the text but not visually aggressive;
- font feels technical/typewritten;
- no external font dependency is introduced;
- no canonical CSL/content files are modified;
- only CSS/static print assets are touched unless there is a specific reason otherwise.

---

## 7. Non-Goals

This FlagFix must not:

- modify original PureDhamma text;
- modify translations;
- modify CSL canonical content;
- normalize didactic colors;
- change glossary/audio taxonomy;
- add remote fonts;
- change print margins or page layout globally;
- redesign the entire print theme.

---

## 8. Architectural Note

The print review banner is part of the review artifact layer, not the doctrinal content layer.

It should remain visually distinct from the essay itself. The more it resembles a machine-generated archival stamp, the less likely a human reviewer is to confuse it with source content.

This supports the larger AXIS-NIDDHI principle:

> Preserve first. Review transparently. Improve surgically.
