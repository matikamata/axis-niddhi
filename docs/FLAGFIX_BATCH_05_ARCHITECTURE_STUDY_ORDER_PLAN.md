# FlagFix Batch 05 — Architecture and Study Order

## Purpose

This planning document groups the FlagFix items related to long-term architecture, canonical study order, preservation strategy, and AXIS-Cosmos / study path evolution.

This batch is intentionally planning-first. It should not change production rendering, CSL content, translations, or deployed pages directly.

- #9 — FlagFix 000 AXIS PRESERVATION STRATEGY
- #10 — FlagFix 001 FUTURE PRESERVATION LAYERS

- #17 — FlagFix 008b AXIS COSMOS ORDER GRAPH DISCUSSION

## Core Problem

AXIS-NIDDHI currently has multiple valid orders for the same corpus:

1. PureDhamma.net original website/navigation order.
2. AXIS archive/static site order.
3. Metadata/CSV/pipeline order.
4. Human study order discovered by readers/reviewers.
5. Future pedagogical order generated by AXIS-NAVIGATOR / AXIS-Cosmos.

None of these should be casually overwritten by another.

The future system needs to preserve all orders as explicit, named, auditable paths rather than pretending there is only one canonical sequence.

## Proposed Direction

Create a study path registry model where each order is represented as a traceable path:

- `source_order`
- `archive_order`
- `review_order`
- `beginner_path`
- `conceptual_path`
- `dependency_path`
- `human_curated_path`
- future `axis_cosmos_graph_path`

Each path should contain:

- path id
- path name
- purpose
- provenance
- creator/reviewer
- date
- ordered list of PD#PN values
- optional notes
- validation status

## Non-goals

This batch does not decide the final universal study order.

This batch does not rewrite archive ordering.

This batch does not modify CSL content.

This batch does not introduce AI-generated ordering into production without human review.

## Safety Principles

- Keep PureDhamma.net source provenance visible.
- Preserve existing order as evidence, not as an obstacle.
- Treat alternate orders as layers, not replacements.
- Require human review for public study paths.
- Never hide uncertainty behind polished UI.
